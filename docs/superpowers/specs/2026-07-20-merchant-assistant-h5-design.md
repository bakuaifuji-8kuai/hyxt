# 商户助手 H5 设计文档

## 1. 项目概述

### 1.1 背景
商户助手 H5 是恒伟商业会员平台的移动端应用，面向入驻商户（门店老板/店员），提供经营看板、优惠券核销、营销活动、公告通知、店铺设置等功能。

### 1.2 目标用户
- 入驻商户的门店负责人
- 门店店员/收银员

### 1.3 核心场景
- 查看门店经营数据（销售额、订单量、核销数）
- 扫码核销顾客出示的优惠券/团购券
- 查看商场活动并报名参加
- 接收商场运营方通知公告
- 维护门店基础信息

## 2. 技术方案

### 2.1 技术栈
- **框架**：React 18 + React Router 6（复用现有项目）
- **UI 组件**：Ant Design 5（复用现有依赖）
- **移动端适配**：CSS 媒体查询 + viewport 配置
- **状态管理**：复用现有 `store.ts` 模拟数据层
- **路由模式**：HashRouter（兼容静态部署）

### 2.2 路由结构
```
/mobile                     # 移动端入口
├── /mobile/home            # 首页（经营看板）
├── /mobile/verify          # 核销中心
├── /mobile/verify-record   # 核销记录
├── /mobile/activities      # 营销活动列表
├── /mobile/activity/:id    # 活动详情
├── /mobile/notices         # 公告通知列表
├── /mobile/notice/:id      # 通知详情
├── /mobile/shop            # 店铺设置
└── /mobile/login           # 登录页
```

### 2.3 数据复用
完全复用现有 mock 数据层（`store.ts`），关键模块：
- `bapp/sales-stats`：销售数据统计
- `bapp/verify-stats`：核销数据统计
- `bapp/coupon-verify`：优惠券核销记录
- `bapp/order-verify`：订单核销记录
- `bapp/activity-verify`：活动核销记录
- `bapp/shop-notice`：店铺通知
- `bapp/shop-info`：店铺信息
- `marketing/campaigns`：营销活动

## 3. 页面设计

### 3.1 登录页（/mobile/login）

**功能**：商户身份验证
**交互设计**：
- 顶部 Logo + 应用名称
- 手机号输入框
- 验证码输入框（模拟：任意验证码可登录）
- "登录"按钮
- 底部版权信息

**数据**：
- 登录成功后存储 `mobileToken` 和 `shopInfo` 到 localStorage
- 模拟商户：任意手机号可登录，自动关联一家门店

### 3.2 首页（/mobile/home）

**功能**：经营看板，展示门店核心数据
**交互设计**：
- 顶部：门店名称 + 日期
- 核心数据卡片：
  - 今日销售额（大字突出，单位元）
  - 今日订单数
  - 今日核销笔数
  - 今日发放优惠券数
- 近7天销售趋势图（简化折线图）
- 快捷入口（2x2 网格）：
  - 扫码核销
  - 发放优惠券
  - 查看活动
  - 店铺设置

**数据来源**：
- `bapp/sales-stats`：销售数据
- `bapp/verify-stats`：核销数据
- `bapp/coupon-issue`：发券数据

### 3.3 核销中心（/mobile/verify）

**功能**：扫码/手动核销优惠券、团购券、订单
**交互设计**：
- 大按钮"扫码核销"（居中，蓝色，宽 80%）
- 手动输入区：
  - 券码输入框
  - "查询并核销"按钮
- 核销结果弹窗：
  - 成功：显示券码、优惠金额、会员信息
  - 失败：显示失败原因（已核销、已过期、不存在等）
- 最近核销记录（卡片列表，显示最近 5 条）

**核销流程**：
1. 扫码或输入券码
2. 调用 API 查询券码信息
3. 展示券码详情供确认
4. 确认核销
5. 显示核销结果

**数据来源**：
- `bapp/coupon-verify`：优惠券核销记录
- `bapp/order-verify`：订单核销记录
- `bapp/group-verify`：团购核销记录
- `bapp/activity-verify`：活动核销记录

### 3.4 核销记录（/mobile/verify-record）

**功能**：查看历史核销记录
**交互设计**：
- 顶部筛选：日期范围、核销类型
- 记录列表（无限滚动）：
  - 券码/订单号
  - 核销类型（优惠券/团购/订单/活动）
  - 会员信息
  - 核销时间
  - 核销状态

### 3.5 营销活动列表（/mobile/activities）

**功能**：查看商场发布的营销活动
**交互设计**：
- 活动卡片列表：
  - 活动封面图
  - 活动名称
  - 活动时间
  - 报名状态标签（未报名/已报名/已结束）
- 点击卡片进入详情页

**数据来源**：
- `marketing/campaigns`：营销活动列表

### 3.6 活动详情（/mobile/activity/:id）

**功能**：查看活动详情并报名
**交互设计**：
- 活动封面图
- 活动名称
- 活动时间
- 活动介绍（富文本）
- 参与商户列表
- "我要报名"按钮（已报名显示"已报名"）
- 报名弹窗：确认报名信息

### 3.7 公告通知列表（/mobile/notices）

**功能**：接收商场运营方通知
**交互设计**：
- 通知卡片列表：
  - 通知图标（系统/活动/合同）
  - 通知标题
  - 发布时间
  - 已读/未读状态
- 点击卡片进入详情页

**数据来源**：
- `bapp/shop-notice`：店铺通知

### 3.8 通知详情（/mobile/notice/:id）

**功能**：查看通知详情
**交互设计**：
- 通知标题
- 发布时间
- 通知正文
- 返回按钮

### 3.9 店铺设置（/mobile/shop）

**功能**：维护门店基础信息
**交互设计**：
- 门店头像 + 名称
- 门店分类
- 营业时间
- 联系电话
- 合同到期时间（临近提醒）
- 账号信息（手机号）

**数据来源**：
- `bapp/shop-info`：店铺信息
- `merchant/list`：商户信息

## 6. 注意事项

1. **扫码功能**：H5 环境下调用相机需要 HTTPS，开发环境使用手动输入替代
2. **数据隔离**：商户只能查看自己门店的数据
3. **离线场景**：核销记录支持本地缓存，网络恢复后同步
4. **兼容性**：适配 iOS Safari 和 Android 微信内置浏览器
